## 网友提问(一生一世)

1、业务中台流程差异如何实现，有几种实现方式？

2、业务中台规则差异如何实现，有几种实现方式？

3、业务中台原子服务功能差异，有几种实现方式？

4、业务中台业务模型差异如何实现？

5、业务前台如何快捷接入业务中台？

6、业务中台业务部署架构和业务前台部署在一起还是分开部署，如何部署？

7、建业务中台的时候，数据中台是不是也要一起建立？如果要一起建立，如何才能做到业务中台数据实时同步数据给数据中台？

## 心得

1. 没有必要为了做中台而做中台
2. 中台是"服务"业务的 
3. 中台的矛盾是：端到端交付和复用之间的矛盾。 
4. 哲学家说过：人需要先进行实践才能理解抽象。 
5. 中台的责任边界比技术更重要。 
6. 任何技术都有背后的代价。
7. 技术问题最终还是要回归到业务问题上。
