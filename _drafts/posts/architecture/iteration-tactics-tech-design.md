## 要点

- 区分设计态和完成态：设计态用完即丢，完成态使用**活文档**
- 文本化：容易版本管理和形式化
- 形式化：精确定义，容易使用它做代码生成
- 可被反合：能对比设计和实现的差异
- 强制和推荐：只做关键设计
  - 数据库设计
  - API 设计
  - 领域模型设计
  - 枚举设计
  - 交互流程设计
